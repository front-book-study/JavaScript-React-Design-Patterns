### GoF 디자인 패턴

- 객체 지향 설계의 문제나 이슈에 초점을 맞춤
- 어디에 적용할 수 있는지. 다른 설계 조건에도 부합하는지. 사용함으로써 얻는 장단점 & 결과. 구현을 위한 예시코드가 포함됨.

### 3가지 종류

- 생성 패턴 : 객체를 생성하는 방법에 중점.
  - 5개 : 팩토리 / 추상 / 프로토타입 / 싱글톤 / 빌더 패턴이 있음.
- 구조 패턴 : 객체의 구성과 각 객체 간의 관계를 인식하는 방법에 중점. 시스템의 한부분에 변경사항이 있더라고 다른 부분에 영향이 가지 않도록 도와줌.
  - 7개 : 데코레이터 / 퍼사드 / 플레이웨이트 / 어댑터 / 프록시 / 브릿지 / 컴포지트
- 행위 패턴 : 시스템 내의 객체 간 커뮤니케이션을 개선 및 간소화하는 방법에 중점. 객체 간의 공통적인 커뮤니케이션 패턴을 감지하고 책임을 분배함으로써 커뮤니케이션의 유연성을 높이고 객체의 행위를 추상화.
  - 11개 : 이터레이터 / 중재자 / 관찰자 / 방문자 / 인터프리터 / 템플릿 메서드 / 책임 연쇄 / 커맨드 / 메멘토 / 상태 / 전략
