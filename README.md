## 📣 진행 방식 📣

<div align="left">

- 스터디는 주 1회 (일: 2:00PM ) 진행됩니다.
- 스터디는 책의 챕터를 기준으로 분류됩니다. 스터디원 모두는 스터디 진행 전까지 해당되는 챕터를 읽어옵니다.
- 발표자, 서기는 당일 랜덤으로 지정됩니다.
- 스터디원 모두는 스터디 진행 전까지 내용 정리를 진행합니다.
  - 정리본은 markdown으로 작성하고 제목은 담당자의 이름입니다.
  - 챕터 명으로 만들어진 폴더에 업로드합니다. (ex. [2장] 타입/황정현.md) 그리고 README에 업로드된 정리본을 연결해줍니다.
  - branch : main
  - commit message : 📚: [2장] 타입 - 황정현
- 스터디원 모두는 스터디 진행 전까지 해당 주차의 내용에 관한 문제 + 디자인 패턴을 적용한 예시코드를 준비합니다.
  - [GitHub issue](https://github.com/front-book-study/JavaScript-React-Design-Patterns/issues)로 해당 챕터의 label을 적용하여 문제를 작성합니다. 양식은 제공되는 템플릿을 준수합니다.
  - 문제는 스터디 진행 전까지 등록합니다.
  - 문제 출제자를 제외한 나머지 스터디원은 발표 이후 10분간 문제에 대한 답안을 답변으로 작성합니다.
