### 디자인 패턴 구조

- 컨텍스트 : 패턴이 적용되는 상황
- 집중 목표 : 패턴을 적용할 떄 고려해야 하는 목표
- 구성 : 제시한 상황과 고려하는 점들을 해결하는 구성

### 구성요소

- 이름
- 설명
- 컨텍스트 개요
- 문제 제시
- 해결 방법
- 설계 내용
- 구현 방법
- 시각적 설명
- 예제
- 필수 연계 : 해당 패턴과 함께 사용하면 좋은 패턴에 대한 내용
- 관계성 : 다른 패턴과의 관계 및 유사점에 관한 내용
- 알려진 용도
- 토론

### 새로운 디자인 패턴 만들기

- 얼마나 실용적인가?
- 모범 사례 염두해두기
- 사용자에게 솔직해야 함
- 독창성은 패턴 설계의 핵심이 아님 : 다른 패턴과 조급 겹치는 부분이 있어도 상관없음. 넓게 적용할 수 있는 패턴이라면 유용한 패턴으로 인정받을 수 있음
- 훌륭한 예시 필요
