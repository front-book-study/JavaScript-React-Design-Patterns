### 디자인 패턴의 역사

- 건축가에 의해서 처음 등장. 반복되는 디자인 구조를 정리.
- 개발에도 디자인 패턴을 정립 → GoF (Gang of Four)

### 패턴이란

- 반복되는 문제와 주제에 적용할 수 있는 재사용 가능한 템플릿
- 장점 : 검증됨 / 쉽게 재사용 / 알아보기 쉬움 / 사소한 실수로 생길 수 있는 큰 문제 방지 / 종합적인 해결책 제시 / 의사소통 원활 → 공통된 어휘 사용 / 선순환 유지

### 일상 활용 사례

- 공급자 패턴 (context provider)
- 해결하는 문제 : 프롤 드릴링
- context api 및 여러 상태관리 라이브러리들이 차용
