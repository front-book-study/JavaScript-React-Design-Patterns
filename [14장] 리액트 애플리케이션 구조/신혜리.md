## 리액트 애플리케이션 구조

- 모듈, 기능 또는 경로별 그룹화
  - 장점 : 모듈에 변경사항이 있을 때 관련된 모든 파일이 같은 폴더에 모여 있어 변경사항이 특정 코드 영역으로 제한
  - 단점 : 공통적으로 사용되는 컴포넌트, 로직, 스타일을 주기적으로 파악해야만 중복을 피하고 일관성과 재사용성을 높일 수 있음

- 파일 유형별 그룹화
  - 장점 : 
    - 표준 구조 재사용 - 여러 프로젝트에서 동일하게 사용할 수 있는 표준적인 구조
    - 온보딩이 빠름 
    - 공통 컴포넌트 및 스타일 변경 용이
  - 단점 : 
    - 모듈 수정 시 여러 폴더 수정 필요
    - 파일 찾기 어려움

- 도메인 + 공통 컴포넌트
  - 공통 : 파일 유형별 그룹화 (components, hooks, utils ...)
  - 도메인 : 도메인별 그룹화
  ** 3-4 단계 이상의 깊은 중첩은 피하는 것이 좋음
